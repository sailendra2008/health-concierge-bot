[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's walk through the architecture diagram showing how request flows work.
First, the user query enters the Security Checkpoint function node which checks for prompt injections, scrubs sensitive PII like phone numbers or emails, and verifies that the patient has provided consent.
If a prompt injection is detected, the workflow routes along the SECURITY_EVENT path directly to the security alert node to reject the input.
Normally, it goes along the DEFAULT_ROUTE to the orchestrator_agent.
The orchestrator_agent is the coordinator, using AgentTool to delegate specialized tasks to the medical_explainer_agent for analyzing lab values, or to the reminder_agent for scheduling daily medications.
Both specialized agents call our local stdio MCP Server to run clinical tools.
After the orchestrator completes, the post_orchestration_node checks if a reminder is being scheduled, routing to the human_verification node to trigger a human-in-the-loop approval.
Once approved, the final response is shown to the user.
